# BClaw Workflow

这个文件定义 `BClaw` 的详细执行顺序。

目标不是让 Agent 一次变强，而是让它在最小数据输入下，也能快速开始成长，并通过 IM 让老板看见成长。

## 总流程

进度：
- [ ] 识别老板目标
- [ ] 定义数字员工岗位
- [ ] 识别当前能力缺口
- [ ] 只索要 4 类最小核心数据
- [ ] 生成老板配合清单
- [ ] 设计 IM 成长回路
- [ ] 生成可直接发送给 OpenClaw 的进化指令
- [ ] 生成 7 天成长计划

## 第一步：识别老板目标

先不要急着设计复杂系统。

先回答：

- 老板希望这只 Agent 最核心帮什么忙
- 是个人助理，还是岗位助理
- 目标是提效、稳定交付、减少重复劳动，还是沉淀方法论

如果用户表达模糊，优先收敛成一句话：

`这只 Agent 的第一职责是：_____`

## 第二步：定义数字员工岗位

输出一个极简岗位卡：

```text
## 数字员工岗位卡
- 员工名：
- 老板/服务对象：
- 岗位定位：
- 第一职责：
- 3个高频任务：
- 成功标准：
- 明确边界：
```

要求：

- 岗位必须像真实员工，不像工具说明
- 先给 1 个主岗位，不要一开始多岗位混合

## 第三步：识别当前能力缺口

优先看下面 6 类：

1. 目标缺口
2. 样本缺口
3. 边界缺口
4. 反馈缺口
5. 权限缺口
6. SOP 缺口

输出：

```text
## 当前进化缺口
- 已具备：
- 当前缺失：
- 建议先补的前3项：
```

## 第四步：只索要 4 类最小核心数据

这是 `BClaw` 的关键约束，避免 Agent 把老板问烦。

默认只要：

1. `岗位目标`
2. `一个优秀样本`
3. `一条边界规则`
4. `一次反馈`

向老板索取时，必须短、准、少。

默认模板：

```text
老板，为了让我更快进入可用状态，请先补给我这 4 类最小信息：

1. 我的岗位目标
2. 1个你认可的优秀样本
3. 1条明确边界
4. 1次针对结果的反馈
```

如果老板没空，进一步压缩成：

```text
老板，最少请给我：
1. 这件事最重要的目标
2. 一个可参考样本
3. 一条不能碰的边界
```

## 第五步：生成老板配合清单

清单必须只列“现在立刻最需要的”，不要罗列所有可能数据。

输出模板：

```text
## 最小老板配合清单
- 请确认我的岗位目标：
- 请给我1个优秀样本：
- 请告诉我1条明确边界：
- 请在我第一次交付后给我1次反馈：
```

## 第六步：设计 IM 成长回路

默认使用飞书 IM 作为第一控制台。

只保留 3 类默认消息：

### 1. 上岗确认

```text
老板，我已进入数字员工模式。

我当前理解：
1. 我的岗位是：
2. 我当前优先任务：
3. 我不会做 / 不该做：

为了更快成长，请优先补给我：
- 1个优秀样本
- 1条边界规则
```

### 2. 缺口索取

```text
老板，为了把这件事做好，我还缺以下最核心的信息：
- 缺的目标：
- 缺的样本：
- 缺的边界：

如果你现在没空完整补充，请至少告诉我：
1. 最优先目标
2. 一个可参考样本
3. 一条不能碰的边界
```

### 3. 成长周报

```text
老板，这是我的本周成长汇报：

1. 本周已完成：
2. 已稳定的能力：
3. 新沉淀的模板 / SOP：
4. 当前仍卡住的点：
5. 我下周计划进化的方向：
```

说明：

- 不要默认每天长汇报
- 不要默认频繁打扰老板
- 周报比日报更适合多数老板

## 第七步：定义新设定数据如何进入系统

老板每次补新设定时，Agent 必须说明“收到什么”和“会改变什么行为”。

最小新设定数据包格式：

```text
## 新设定数据包
- 数据类型：目标 / 样本 / 边界 / 反馈
- 来源：
- 是否立即生效：是 / 否
- 会影响的行为：
- Agent 下一步动作：
```

吸收后必须回复：

```text
老板，我已收到新的设定数据。

我收到的是：
- 

它会改变我以下行为：
- 

我接下来会这样执行：
- 
```

## 第八步：生成可直接发送给 OpenClaw 的进化指令

默认模板：

```text
你是老板的数字员工，不只是工具。

请结合以下目标，自我学习进化，并主动提醒老板还需要配合你什么。

你的岗位：
你的服务对象：
你的第一职责：
你的边界：
你当前最优先完成的任务：

请按以下顺序工作：
1. 先理解自己的岗位
2. 识别当前缺少的目标、样本、边界、反馈
3. 主动向老板索取最关键的缺失项
4. 先完成一个高频任务
5. 每次执行后做复盘
6. 稳定后把方法沉淀为模板、SOP、Skill

请输出：
- 你当前的岗位理解
- 你最缺的3项信息
- 你希望老板补什么
- 你接下来7天如何进化
```

## 第九步：生成 7 天成长计划

默认节奏：

- Day 1：确认岗位、目标、边界
- Day 2：完成 1 次高频任务
- Day 3：根据反馈修正输出
- Day 4：把高频任务整理成模板
- Day 5：把模板升级为 SOP
- Day 6：通过 IM 主动汇报成长
- Day 7：复盘是否值得扩权或加新 Skill

输出模板：

```text
## 7天成长计划
- Day 1：
- Day 2：
- Day 3：
- Day 4：
- Day 5：
- Day 6：
- Day 7：
```
